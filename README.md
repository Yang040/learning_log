### 2. Python基础（已学）

**6种数据类型**：int、float、str、bool、list、dict

**if/elif/else**：条件判断，处理多种分支

**assert**：断言，条件为假时抛出异常，用于底线校验

**assert vs if/else**：
- assert：验证“必须为真”，条件为假中断程序 → 单个用例最终验证点
- if/else：处理“两种可能”，条件为假继续执行 → 流程控制、结果统计

**for循环**：遍历列表、range()生成序列、批量执行用例

**while循环**：条件循环，适用重试机制

**函数封装**：
- def 函数名(参数): 定义
- return 返回结果
- 函数负责返回结果，判断对错交给外层（测试逻辑与业务逻辑分离）

## 五、重要学习原则

1. **只学该学的**：Python只学到函数/异常处理，不碰类/继承/多线程；算法只刷LeetCode简单
2. **每周必有产出**：代码文件、用例文档、Bug报告
3. **AI辅助但不依赖**：每行代码必须能讲清楚
4. **独立练习**：先自己做，做完再看参考，不做出来可以问，但先试
5. **断言和判断各司其职**：函数返回结果，外层if/else统计，assert守底线

问题1：@pytest.fixture 这行是干什么的？我的回答“确定夹具函数在哪”正确表述@pytest.fixture 是一个装饰器，把函数注册为pytest的fixture，让测试函数可以通过参数名来调用它。
问题2：valid_user 参数为什么能拿到数据？你的回答：“valid_user 执行获得了 return 的值”

✅ 正确。 这就是 fixture 的核心机制：pytest 看到测试函数参数名 valid_user，就去查找同名的 fixture 函数，执行它，把返回值传给测试函数
问题3：yield 前面和后面的代码分别什么时候执行？

你的回答：“yield 前面的代码在程序开始时执行，后面的程序完成后执行”

方向对，但细节有偏差。 不是“程序开始时”和“程序完成后”，而是：

yield 前的代码：每个使用这个fixture的测试函数执行前执行

yield 后的代码：测试函数执行结束后执行

正确表述：yield 把一个 fixture 分成了“准备阶段”和“清理阶段”。准备阶段在测试前执行，清理阶段在测试后执行，无论测试通过还是失败都会执行。
